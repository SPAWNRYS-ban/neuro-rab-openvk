# Image Handling и Vision API Update

## Краткое резюме

Был добавлен полный набор инструментов для обработки изображений с защитой от Claude API 400 ошибок:

✅ **Валидация размера изображений** перед отправкой в API  
✅ **Конфигурация Vision API** через переменные окружения  
✅ **Улучшенное логирование** для отладки  
✅ **Graceful fallback** на текстовой анализ при ошибках  
✅ **Полная документация** процесса

---

## Быстрый старт

### 1. Обновите .env файл

Добавьте новые параметры конфигурации:

```bash
# Отключить Vision API если tokenator.cloud его не поддерживает
ENABLE_VISION_API=false

# Или задать лимит размера изображений (в МБ)
VISION_API_MAX_IMAGE_SIZE_MB=10
```

### 2. Варианты конфигурации

**Если Vision API работает:**
```env
ENABLE_VISION_API=true
VISION_API_MAX_IMAGE_SIZE_MB=20
```

**Если Vision API вызывает ошибки:**
```env
ENABLE_VISION_API=false
```

**Если нужна защита от больших изображений:**
```env
ENABLE_VISION_API=true
VISION_API_MAX_IMAGE_SIZE_MB=5
```

## Как это работает

### Процесс обработки изображений

```
Пост с изображением
    ↓
Проверка: ENABLE_VISION_API=true? 
    ├─ false → Пропустить изображение
    └─ true → Скачать изображение
        ↓
    Проверка размера файла < VISION_API_MAX_IMAGE_SIZE_MB?
        ├─ false → Пропустить (логировать "Image too large: X MB exceeds limit of Y MB")
        └─ true → Base64 кодировать
            ↓
        Отправить в Claude Vision API
            ├─ ❌ Ошибка → Fallback на текстовой анализ (логировать ошибку)
            └─ ✅ Успех → Использовать результат анализа изображения
```

### Логирование

Бот теперь логирует:

**При успехе:**
```
✅ Successfully processed image from comment, size: 1048576 bytes
✅ Vision analysis completed successfully
```

**При ошибке размера:**
```
Failed to process image from comment: Image too large: 25.5 MB exceeds limit of 20 MB (will skip this image)
```

**При отключении Vision API:**
```
Vision API is disabled in config
```

**При ошибке API:**
```
❌ Vision analysis failed (Request failed), will fallback to text-only
```

## Решение проблем

### Проблема: 400 Bad Request от Claude API

**Решение:**
```env
ENABLE_VISION_API=false
```
Это отключает Vision API полностью и использует только текстовой анализ.

### Проблема: "Image too large: 25.5 MB exceeds limit"

**Решение 1:** Увеличить лимит
```env
VISION_API_MAX_IMAGE_SIZE_MB=30
```

**Решение 2:** Уменьшить лимит (если API имеет внутренние ограничения)
```env
VISION_API_MAX_IMAGE_SIZE_MB=5
```

### Проблема: Vision API иногда не работает

**Решение:** Это нормально! Бот автоматически падбэчится на текстовой анализ. Смотрите логи для деталей.

## Технические детали

### Размер base64-encoded данных

Важно помнить, что при base64 кодировании размер файла увеличивается на ~33%:

```
100 МБ изображение → ~133 МБ base64 данных
```

Поэтому если API имеет лимит на запрос, лучше установить `VISION_API_MAX_IMAGE_SIZE_MB` на значение ниже реального лимита API.

### Поддерживаемые форматы изображений

Автоматически распознаются:
- PNG (signature: `\x89PNG\r\n\x1a\n`)
- JPEG (signature: `\xff\xd8\xff`)
- WebP (signature: `RIFF...WEBP`)
- GIF (signature: `GIF87a` или `GIF89a`)

Другие форматы обрабатываются как JPEG (fallback).
